## Cos'è l'incapsulamento?

L'**incapsulamento** è uno dei principi fondamentali dell'OOP. Significa nascondere i dettagli interni di un oggetto e fornire solo un'interfaccia pubblica controllata per interagire con esso.

In pratica: i dati (campi) di una classe vengono dichiarati **privati**, e si accede a essi solo tramite metodi **pubblici** (getter e setter).

Vantaggi:
- Protezione dei dati da modifiche accidentali
- Possibilità di validare i valori prima di modificarli
- Libertà di cambiare l'implementazione interna senza rompere il codice esterno

## I modificatori di accesso

In C++ ci sono tre livelli di accesso:

| Modificatore | Accessibile da |
|---|---|
| `public` | ovunque |
| `private` | solo dall'interno della classe |
| `protected` | dalla classe e dalle sue derivate (ereditarietà) |

## Esempio senza incapsulamento

```cpp
class Eta {
public:
    int valore;   // pubblico: chiunque può modificarlo
};

int main() {
    Eta e;
    e.valore = -5;   // nessuno impedisce valori assurdi!
    e.valore = 1000; // nessun controllo!
    return 0;
}
```

## Esempio con incapsulamento

```cpp
class Eta {
private:
    int valore;   // privato: nessuno può accedervi direttamente

public:
    // Setter: valida il valore
    void setValore(int v) {
        if (v >= 0 && v <= 120) {
            valore = v;
        } else {
            cout << "Eta non valida!" << endl;
        }
    }

    // Getter: restituisce il valore
    int getValore() const {
        return valore;
    }
};

int main() {
    Eta e;
    e.setValore(16);      // OK
    e.setValore(-5);      // "Eta non valida!"
    e.setValore(1000);    // "Eta non valida!"

    cout << e.getValore() << endl;   // 16

    // e.valore = 5;  // ERRORE: valore è privato!

    return 0;
}
```

## Un esempio completo: classe Persona

```cpp
#include <iostream>
#include <string>
using namespace std;

class Persona {
private:
    string nome;
    int eta;
    string email;

public:
    // Costruttore
    Persona(string n, int e, string em) {
        nome = n;
        setEta(e);     // usa il setter per validare
        email = em;
    }

    // Getter
    string getNome() const { return nome; }
    int getEta() const { return eta; }
    string getEmail() const { return email; }

    // Setter con validazione
    void setEta(int e) {
        if (e >= 0 && e <= 150) {
            eta = e;
        } else {
            cout << "Eta non valida!" << endl;
            eta = 0;
        }
    }

    void setEmail(string em) {
        // Controllo minimo: deve contenere @
        if (em.find('@') != string::npos) {
            email = em;
        } else {
            cout << "Email non valida!" << endl;
        }
    }

    // Metodo pubblico per stampa
    void stampa() const {
        cout << "Nome: " << nome << endl;
        cout << "Eta: " << eta << endl;
        cout << "Email: " << email << endl;
    }
};

int main() {
    Persona p("Alice", 16, "alice@esempio.it");
    p.stampa();

    cout << endl;
    p.setEta(17);       // OK
    p.setEmail("non-una-email");  // Email non valida!
    p.setEmail("alice.nuovo@esempio.it");  // OK

    cout << "Eta: " << p.getEta() << endl;
    cout << "Email: " << p.getEmail() << endl;

    return 0;
}
```

## Private per default nelle classi

Ricorda: in una `class`, i membri sono `private` per default (al contrario della `struct`):

```cpp
class Esempio {
    int x;   // privato per default!

public:
    int y;   // pubblico
};
```

È buona pratica specificare sempre esplicitamente `public:` e `private:` per chiarezza.

## Incapsulamento e modifica dell'implementazione

L'incapsulamento ti permette di cambiare come funziona internamente una classe senza cambiare come si usa:

```cpp
// Versione 1: temperature in Celsius
class Termometro {
private:
    double celsius;
public:
    void setTemperatura(double t) { celsius = t; }
    double getTemperatura() const { return celsius; }
};

// Versione 2: memorizza in Kelvin internamente
// Ma l'interfaccia pubblica è identica!
class Termometro {
private:
    double kelvin;  // cambiato internamente
public:
    void setTemperatura(double celsius) { kelvin = celsius + 273.15; }
    double getTemperatura() const { return kelvin - 273.15; }
    // Chi usa questa classe non sa (e non deve sapere) del cambio!
};
```

Questa è la forza dell'incapsulamento: il codice che usa la classe non deve cambiare.
