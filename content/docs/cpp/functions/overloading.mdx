## Cos'è l'overloading?

L'**overloading** (sovraccarico) di funzioni permette di definire più funzioni con lo **stesso nome** ma con **parametri diversi** (numero o tipo diverso). Il compilatore sceglie automaticamente quale versione chiamare in base agli argomenti forniti.

Questo è una caratteristica del C++ non presente in C.

## Esempio base

```cpp
void stampa(int x) {
    cout << "Intero: " << x << endl;
}

void stampa(double x) {
    cout << "Decimale: " << x << endl;
}

void stampa(string s) {
    cout << "Stringa: " << s << endl;
}

int main() {
    stampa(42);           // chiama stampa(int)
    stampa(3.14);         // chiama stampa(double)
    stampa("Ciao");       // chiama stampa(string)
    return 0;
}
```

Output:

```
Intero: 42
Decimale: 3.14
Stringa: Ciao
```

Il compilatore distingue le funzioni dalla **firma**: il nome più i tipi dei parametri.

## Overloading con numero diverso di parametri

Puoi anche sovraccaricare con un numero diverso di parametri:

```cpp
int somma(int a, int b) {
    return a + b;
}

int somma(int a, int b, int c) {
    return a + b + c;
}

int somma(int a, int b, int c, int d) {
    return a + b + c + d;
}

int main() {
    cout << somma(1, 2) << endl;        // 3
    cout << somma(1, 2, 3) << endl;     // 6
    cout << somma(1, 2, 3, 4) << endl;  // 10
    return 0;
}
```

## Esempio: area di figure geometriche

Un uso classico dell'overloading è calcolare l'area di forme diverse con la stessa funzione `area`:

```cpp
#include <iostream>
using namespace std;

const double PI = 3.14159265358979;

// Area del cerchio
double area(double raggio) {
    return PI * raggio * raggio;
}

// Area del rettangolo
double area(double base, double altezza) {
    return base * altezza;
}

// Area del triangolo
double area(double base, double altezza, bool triangolo) {
    return (base * altezza) / 2;
}

int main() {
    cout << "Area cerchio (r=5): " << area(5.0) << endl;
    cout << "Area rettangolo (4x6): " << area(4.0, 6.0) << endl;
    cout << "Area triangolo (3x8): " << area(3.0, 8.0, true) << endl;
    return 0;
}
```

## Quando il compilatore sceglie la versione sbagliata

A volte il compilatore può essere ambiguo. Pensa a questo esempio:

```cpp
void f(int x) { cout << "int" << endl; }
void f(double x) { cout << "double" << endl; }

int main() {
    f(5);     // chiama f(int)
    f(5.0);   // chiama f(double)
    f(5.0f);  // chiama f(double) (float viene promosso a double)
    // f(5L);  // potrebbe essere ambiguo su alcuni compilatori
    return 0;
}
```

## Cosa NON è overloading

Il tipo di ritorno da solo **non distingue** le funzioni overloaded. Questo causa un errore:

```cpp
int calcola(int x) { return x * 2; }
double calcola(int x) { return x * 2.0; }  // ERRORE: firma identica!
```

Il compilatore non può scegliere la versione corretta basandosi solo su dove viene usato il risultato.

## Overloading vs valori di default

Spesso si può ottenere lo stesso effetto sia con l'overloading sia con i parametri di default:

```cpp
// Con overloading
void saluta() { cout << "Ciao, ospite!" << endl; }
void saluta(string nome) { cout << "Ciao, " << nome << "!" << endl; }

// Con parametro di default
void saluta(string nome = "ospite") {
    cout << "Ciao, " << nome << "!" << endl;
}
```

Usa i parametri di default quando la logica è la stessa con valori diversi; usa l'overloading quando la logica è effettivamente diversa.

## Esempio pratico

```cpp
#include <iostream>
#include <string>
using namespace std;

// Overloading di "confronta"
bool confronta(int a, int b) {
    return a == b;
}

bool confronta(double a, double b) {
    // Per i decimali, confronto con tolleranza
    return (a - b < 0.0001) && (b - a < 0.0001);
}

bool confronta(string a, string b) {
    return a == b;
}

int main() {
    cout << boolalpha;
    cout << confronta(5, 5) << endl;           // true
    cout << confronta(3.14, 3.14) << endl;     // true
    cout << confronta("ciao", "ciao") << endl; // true
    cout << confronta("ciao", "Ciao") << endl; // false
    return 0;
}
```
