## Cos'è la ricorsione?

Una funzione si dice **ricorsiva** quando chiama se stessa. È un concetto potente che permette di risolvere problemi complessi dividendoli in sottoproblemi più semplici.

Ogni funzione ricorsiva deve avere:
1. **Caso base**: la condizione che ferma la ricorsione
2. **Caso ricorsivo**: la chiamata a se stessa con un argomento "più piccolo"

Senza un caso base, la funzione chiamrebbe se stessa all'infinito (come un ciclo infinito) finché Python non esaurisce la memoria.

## Esempio classico: il fattoriale

Il fattoriale di `n` (scritto `n!`) è il prodotto di tutti i numeri da 1 a n:
- `5! = 5 × 4 × 3 × 2 × 1 = 120`
- `3! = 3 × 2 × 1 = 6`
- `1! = 1`
- `0! = 1` (per definizione)

```python
def fattoriale(n):
    # Caso base
    if n == 0 or n == 1:
        return 1
    # Caso ricorsivo
    return n * fattoriale(n - 1)

print(fattoriale(5))   # 120
print(fattoriale(3))   # 6
print(fattoriale(0))   # 1
```

Come funziona `fattoriale(5)`:
```
fattoriale(5)
  5 * fattoriale(4)
       4 * fattoriale(3)
            3 * fattoriale(2)
                 2 * fattoriale(1)
                      1  (caso base!)
```

## Esempio: successione di Fibonacci

La successione di Fibonacci: 0, 1, 1, 2, 3, 5, 8, 13, 21, ...
Ogni numero è la somma dei due precedenti.

```python
def fibonacci(n):
    # Casi base
    if n == 0:
        return 0
    if n == 1:
        return 1
    # Caso ricorsivo
    return fibonacci(n - 1) + fibonacci(n - 2)

for i in range(8):
    print(fibonacci(i), end=" ")
# 0 1 1 2 3 5 8 13
```

## Esempio: somma degli elementi di una lista

```python
def somma_lista(lista):
    # Caso base: lista vuota
    if len(lista) == 0:
        return 0
    # Caso ricorsivo: primo elemento + somma del resto
    return lista[0] + somma_lista(lista[1:])

print(somma_lista([1, 2, 3, 4, 5]))  # 15
```

## Esempio: invertire una stringa

```python
def inverti(testo):
    # Caso base: stringa vuota o un carattere
    if len(testo) <= 1:
        return testo
    # Caso ricorsivo
    return inverti(testo[1:]) + testo[0]

print(inverti("Python"))  # nohtyP
```

## Limite di ricorsione

Python ha un limite di default di circa 1000 chiamate ricorsive. Se lo superi, ottieni un `RecursionError`:

```python
import sys
print(sys.getrecursionlimit())  # 1000

# Puoi aumentarlo, ma non è consigliato:
# sys.setrecursionlimit(10000)
```

## Ricorsione vs iterazione

La ricorsione è elegante ma può essere meno efficiente dell'iterazione per alcuni problemi. Ad esempio, `fibonacci()` ricorsivo è molto lento per grandi valori di `n` perché ricalcola gli stessi valori più volte.

Per questi casi si usano tecniche come la **memoizzazione** (salvare i risultati già calcolati):

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fibonacci_veloce(n):
    if n <= 1:
        return n
    return fibonacci_veloce(n - 1) + fibonacci_veloce(n - 2)

print(fibonacci_veloce(100))  # veloce!
```

## Quando usare la ricorsione?

Usa la ricorsione quando:
- Il problema si decompone naturalmente in sottoproblemi della stessa natura
- Stai lavorando con strutture ad albero o grafi
- Il codice ricorsivo è molto più semplice di quello iterativo

Usa l'iterazione per problemi semplici dove la ricorsione sarebbe eccessiva.
